## 5. 建议发布流程

建议每次发布前至少完成以下步骤：

1. 更新版本号
2. 整理 README 与发布说明
3. 运行核心 targeted tests
4. 运行全量 `npm test`
5. 确认未把 `.debug/`、本地监控脚本、临时计划文件等误推到远端

常用验证命令：

```powershell
npm test
```

如只验证核心接码与恢复链路，可优先跑：

```powershell
node --test tests/phone-verification-flow.test.js
node --test tests/signup-page-verification-error-state.test.js
node --test tests/background-free-reusable-phone-country.test.js
```

## 6. 与上游同步建议

当前仍然建议保留“按补丁移植”的节奏，而不是整批硬覆盖：

1. 先比对上游目标 tag / 分支
2. 只挑有价值的修复点
3. 先补失败测试
4. 跑红灯
5. 最小实现
6. 跑 targeted tests
7. 最后全量 `npm test`

这样可以避免本地结构化错误语义、运行态字段与侧边栏扩展被误覆盖。
